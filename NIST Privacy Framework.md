- This framework follows the same overall design as the [[NIST Cybersecurity Framework]] and [[NIST Risk Management Framework]]
- This framework has three main components:

## Core
- The core describes the set of functions and activities that should be included in the privacy program 
- The core breaks down into 5 functional areas:
	- Identify - Inventory and mapping, risk assessment, data processing ecosystem
	- Govern - governance policies, processes and procedures, risk management strategy, awareness and training, monitoring and review
	- Control - data processing policies, procedures and processes
	- Communicate - communication policies, data processing awareness 
	- Protect - data protection policies, identity management, authentication, access control, data security, maintenance, protective technology 

## Profiles 
- Allow organisations to select elements from the Core for prioritisation
- The two identified profiles are:
	- *Current Profile* - describes the current state of privacy operations
	- *Target Profile* - describes the desired state of privacy operations

## Implementation Tiers
- A methods of assessing the maturity of the privacy program 
- There are 4 tiers
	- Partial 
	- Risk Informed 
	- Repeatable 
	- Adaptive
